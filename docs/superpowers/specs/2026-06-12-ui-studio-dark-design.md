# 鹿绘AI UI/交互优化：制片厂深黑方案

## 概述

将鹿绘AI的视觉风格从当前的「通用暗色+紫色强调」升级为「制片厂深黑+桔黄唯一强调色」，同时重组流程中心的按钮体系，解决用户无法区分各推理/同步按钮作用范围的核心痛点。

## 决策记录

| 决策 | 选项 | 选择 | 原因 |
|---|---|---|---|
| 视觉方向 | A 精修暗色 / B 制片厂深黑 / C 明快工作台 | B | 影视后期工具气质，与产品定位（动画制片管线）一致 |
| 强调色 | 保留紫 / 换桔黄 / 其他 | 桔黄 `#F5A623` | 纯正制片厂路线（达芬奇/剪映风） |
| 按钮体系 | 现状散落 / 两层收敛 | 两层：一个主按钮 + 高级下拉 | 消除误导，日常只需一个按钮 |
| 范围 | 仅交互 / 交互+全站视觉 | 交互+全站视觉 | 用户明确要求 |

## §1 视觉系统

### 色板

| 用途 | 色值 | 说明 |
|---|---|---|
| 底色 | `#0D0D0F` | 近纯黑 |
| 面板/卡片 | `#1A1A1E` | 微灰浮层 |
| 边框/分割 | `#2A2A2E` | 低对比分隔 |
| 正文 | `#E8E8EC` | 高可读性白 |
| 次要文字 | `#8A8A8E` | 标签/说明 |
| 强调色（唯一） | `#F5A623` | 桔黄，行动按钮/激活态/进度 |
| 强调悬停 | `#D4890A` | 按下/hover 深一档 |
| 成功 | `#22C55E` | 语义色不变 |
| 错误 | `#EF4444` | 语义色不变 |
| 警告 | `#EAB308` | 语义色不变 |

### 排版

- 字体：保留 Inter / 系统无衬线
- 标题权重：`font-medium`（从 `font-semibold` 降，低对比更专业）
- 阶段编号：`font-variant-numeric: tabular-nums`，强化时间线感

### 组件风格

- **按钮**：主按钮桔黄底/黑字；次要按钮透明底/桔黄描边；幽灵按钮纯文字桔黄
- **卡片**：`border-radius: 4px`（从 8px 收紧）；1px `#2A2A2E` 边框
- **Badge/状态**：低饱和底色 + 桔黄或语义色文字（不再用紫/蓝背景填充）
- **侧边栏选中态**：左边框桔黄 2px + 文字白色（替代紫色背景高亮）
- **输入框焦点态**：桔黄 ring（替代紫色 ring）

## §2 按钮体系重组

### 核心原则

一个主按钮走全流程，高级操作折叠不干扰。

### 流程中心主面板

| 位置 | 按钮名 | 作用 | 样式 |
|---|---|---|---|
| 顶部醒目 | **全流程推理** | 分镜拆解→故事板→视频提示词→放入画布 | 桔黄主按钮，大号 |
| 顶部次要 | 同步到画布 | 只把现有数据铺节点，不调 LM | 幽灵按钮 |

### 高级操作（折叠在「更多操作 ▾」下拉菜单里）

- 重跑分镜脚本（从源文重新拆解，不动后续）
- 只推理故事板（不重跑分镜）
- 只推理视频提示词（不动故事板）
- 批量润色提示词（LLM 对现有提示词做精炼）

### 单个 Clip 卡片

- 主操作：「推理」= 该 Clip 故事板→视频提示词（桔黄按钮）
- 次要操作：「润色」= LLM 精炼现有提示词（幽灵按钮）
- 「复制」保留

### 移除/重命名清单

| 旧按钮 | 处置 |
|---|---|
| 一键推理并放入画布 | 删除（被「全流程推理」替代，新版真正包含分镜） |
| 推理故事板+视频并放入画布 | 删除（与上一个重复） |
| 重新拆解 | 移入高级下拉 |
| 批量重新推理 | 移入高级下拉并改名「批量润色提示词」 |
| 同步本集到画布 | 保留为顶部次要按钮「同步到画布」 |
| 重新推理视频提示词（单 Clip） | 改名「润色」移为次要按钮 |

### 进度反馈

- 全流程运行时：主按钮变进度条（`Clip 3/5 · 故事板推理中...`），所有其他按钮 disabled
- 单 Clip 操作时：只该卡片显示 spinner，其余不受影响
- 错误处理：逐 Clip try/catch，某个失败不中断后续；结束后汇报成功/失败数

## §3 全流程推理 Handler

### 逻辑（前端，串调现有 API）

```
for each clip in clips:
  1. POST /workflow/clips/:clipId/storyboard-plan  (分镜拆解)
  2. finalize storyboard prompt locally
  3. POST /workflow/clips/:clipId/seedance-prompt   (视频提示词)
  4. save workflow state
after all clips:
  5. sync episode to canvas (同步节点)
```

与现有 `handleInferBoardsAndVideoToCanvas` 的区别：**Step 1 换成分镜拆解 API（重跑分镜脚本），而非 storyboard-plan（只规划面板）**。现有 handler 跳过了分镜阶段——这就是你被误导的根因。

### 错误处理

- 分镜成功但故事板失败：保留已更新的分镜数据，继续下一个 Clip
- 视频提示词失败：保留故事板结果，继续下一个 Clip
- 全部完成后弹出摘要：「5/5 成功」或「3/5 成功，2 个失败（Clip 02, Clip 04）」

## §5 风险

| 风险 | 缓解 |
|---|---|
| tailwind 全局换色后某些组件对比度不足 | 实施后逐页浏览器检查，低对比处手动加 `text-white` 或调整 |
| 「全流程推理」中间步骤失败丢数据 | 逐 Clip try/catch + 每步成功后立即 save workflow |
| 桔黄与错误色（红）/警告色（黄）在小尺寸 Badge 上视觉混淆 | 错误/警告 Badge 保持语义色底色，桔黄只用于行动/激活态 |
| 用户已形成紫色品牌认知 | 全站一次性切换，不做渐进（避免混搭过渡期更混乱） |

---

## §6 验收标准

1. 全站无紫色残留（grep `7C3AED`/`purple`/`violet` 零命中）
2. 「全流程推理」按钮点击后依次执行分镜→故事板→视频→同步，日志可观测每步
3. 高级下拉包含 4 个单步操作，主面板上不再直接暴露
4. 侧边栏选中项为桔黄左边框样式
5. 主按钮运行时显示进度文字并 disable 其余操作按钮
6. 浏览器逐页检查无明显对比度/可读性回归
