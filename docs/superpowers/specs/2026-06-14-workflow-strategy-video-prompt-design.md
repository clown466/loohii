# 工作流策略与视频提示词推理重构设计（2026-06-14）

## 背景

当前漫剧工作流把“导入/智能拆解”“分镜脚本”“故事板”“视频提示词”“同步画布”等动作混在多个入口里。用户很难判断：

- 小说/剧本导入里的 AI 智能拆解；
- 分镜脚本里的全流程推理、重新拆解；
- 顶部工具栏里的全流程推理、更多操作；

这些按钮分别会改什么数据。实际使用中还暴露了几个质量问题：

- 选择 `Seedance 多参` 后仍显示故事板图、故事板提示词等章节导演板页面。
- 原文事件顺序被模型错排，例如先让 Flora 开口，才处理 Chloe 开枪击杀门缝丧尸。
- 长对白曾被切到相邻 Clip，造成视频生成不连贯。
- 分镜脚本质量不足，15 秒 Clip 只拆出少量镜头，且镜头内容可能为空。
- 视频提示词没有稳定写清角色位置、朝向、状态、持物和穿戴。
- 视频提示词没有持续追踪状态，容易让角色在后续 Clip 中“恢复默认资产图状态”。
- 资产图约束没有被硬性使用，例如戴头盔角色被写成摸头发。

本设计目标是把工作流收束成策略驱动的分步流程，并让视频提示词推理真正读取原文、资产、分镜、全局设定和前文状态。

## 用户确认的决策

1. 采用“策略驱动的分步向导”，不做完整后端 job/stage 状态机重构。
2. 去掉顶部重复按钮：`全流程推理`、`更多操作`、顶部重跑入口。
3. 工作流保留三个清晰主动作：
   - `提取资产并拆解分镜`
   - `重新拆解分镜`
   - `生成视频提示词并同步画布`
4. `Seedance 多参` 不显示故事板图、故事板提示词、故事板生图区域。
5. `章节导演板` 显示故事板/导演板区域。
6. `首帧衔接` 暂不开发，只显示不可用提示。
7. `普通` 模式从有效策略中移除或隐藏。
8. 视频提示词中的角色位置、朝向、状态、持物、穿戴必须动态推理，不能写死示例角色。
9. 对白时机必须遵守原文。原文是 Chloe 开枪后 Flora 才训斥时，不允许把训斥提前到枪响同时。
10. 状态必须跨 beat/clip 延续。视频模型没有记忆，所以每个 clip prompt 都要显式写当前状态。
11. 资产图是硬约束。只有原文或设定明确允许状态变化时，才可写“先脱掉头盔”等状态转换。
12. 分镜脚本本身很重要，需要作为视频提示词质量的核心输入。
13. 当前项目名保持中文。只翻译当前已填入的、会影响推理的全局设定字段。
14. 测试要更直观、更细致，失败时能看到原文片段、期望结果、实际结果和提示词摘录。

## 工作流设计

### 1. 策略选择

全局设定中的生成策略保留两个有效生产策略：

- `章节导演板`：需要故事板/导演板，视频提示词可引用故事板图作为视觉连续性参考。
- `Seedance 多参`：不需要故事板图，视频提示词直接从资产图、分镜脚本、原文上下文和状态账本推理。

`首帧衔接` 保留为不可用状态，展示“暂未开发”。`普通` 不再作为可选生产路径出现。

### 2. 页面显示

`Seedance 多参`：

- 显示资产、分镜脚本、视频提示词区域。
- 隐藏故事板图片列表。
- 隐藏 Clip 故事板提示词。
- 隐藏故事板生图入口。
- 画布同步视频节点时优先连接角色、场景、道具资产图和可用音频。

`章节导演板`：

- 显示资产、分镜脚本、故事板/导演板、视频提示词区域。
- 画布同步视频节点时连接故事板坑位、资产图和音频。

`首帧衔接`：

- 显示不可用说明。
- 不提供生成按钮，避免用户以为链路可用。

### 3. 主动作

第一步：`提取资产并拆解分镜`

- 读取小说/剧本原文、项目全局设定和已有资产。
- 提取/更新角色、场景、道具。
- 生成结构化分镜脚本。
- 这是新导入内容的主入口。

第二步：`重新拆解分镜`

- 用当前资产、原文和全局设定重跑分镜脚本。
- 不直接生成视频提示词。
- 不直接写入画布视频节点。

第三步：`生成视频提示词并同步画布`

- 基于当前资产、分镜脚本、状态账本、全局设定生成视频提示词。
- 按当前策略同步到画布。
- `Seedance 多参` 不创建故事板图链路。
- `章节导演板` 保留故事板参考链路。

## 推理数据流

### 1. 资产优先

视频提示词不能直接从原文跳到最终 prompt。流程必须是：

source text + global settings
→ asset extraction
→ storyboard/script breakdown
→ character state ledger
→ clip video prompt
→ canvas sync

资产中的角色外观、穿戴、持物、道具绑定、场景位置和参考图分析要作为硬约束进入分镜和视频 prompt。

### 2. 分镜脚本

分镜脚本需要表达：

- 事件顺序；
- 镜头目标；
- 角色位置；
- 角色朝向；
- 动作前后因果；
- 对白开始时机；
- 当前持物和穿戴；
- 明确的画面内容。

分镜拆解不能为了“谨慎”把一个完整句子切到下一个 Clip。一个 Clip 的 15 秒容量允许承载完整对白，只要事件顺序和动作节奏合理。

对于用户给出的门缝丧尸例子，期望拆解顺序是：

1. 门缝处丧尸探头。
2. Chloe 开枪，门缝丧尸被击中。
3. Flora 在枪响后训斥。

不能把 Flora 的对白提前到枪响同时，除非原文明确写明对白和枪响同时发生。

### 3. 状态账本

后端推理层需要在生成视频提示词前构建每个主要角色的状态账本。账本按 beat/shot/clip 顺序更新，至少记录：

- 当前屏幕位置或相对位置；
- 面向方向；
- 镜头距离；
- 当前动作状态；
- 当前持物；
- 当前穿戴；
- 污渍、伤势、血迹、湿透等可见状态；
- 与默认资产图不一致的状态；
- 状态变化来源。

状态变化来源只能来自：

- 原文明确描述；
- 分镜脚本明确描述；
- 用户全局设定明确允许；
- 当前资产事实的自然延续。

如果上一个 beat/clip 中角色丢掉枪，则后续每个 clip prompt 都要说明角色未持枪，直到原文或分镜让角色重新拿起枪。如果上一个 beat/clip 中角色摘下头盔，后续也要继续说明未戴头盔，直到状态变回资产图。

## 视频提示词规则

每个 Clip 的最终视频提示词必须包含：

- Clip 目标和事件顺序；
- 每个 beat/shot 的画面内容；
- 角色屏幕位置；
- 角色朝向；
- 角色当前动作；
- 当前持物；
- 当前穿戴；
- 与资产图不一致的状态；
- 对白文本和说话人；
- 对白发生时机；
- 镜头语言；
- 禁止事项。

位置和状态必须动态推理。例如“Flora 在屏幕右侧中景、面向屏幕左侧；Chloe 在屏幕左侧中景、手持霰弹枪”只能作为格式参考，实际内容要由当前原文、分镜和资产决定。

禁止事项包括：

- 不得改变角色身份。
- 不得让资产图中不存在的道具突然出现。
- 不得让角色做与当前穿戴冲突的动作。
- 不得跳过、合并、重排源文本事件。
- 不得提前对白。
- 不得遗漏当前状态。

## 全局设定翻译

实施时做一次性数据翻译，目标是让后续推理更稳定：

- 翻译当前项目中已经填入的 prompt 相关全局设定字段。
- 保持项目名中文。
- 保持用户没有要求翻译的展示性中文名称。
- 已经是英文且可用的负面词、模型参数或固定 prompt 不做无意义重写。

可翻译字段包括但不限于：

- 项目风格；
- 导演说明；
- 角色身份约束；
- 视觉风格补充；
- 分镜规则；
- 视频提示词规则。

## 错误处理

- `Seedance 多参` 已选但没有资产：视频提示词生成按钮提示先提取/确认资产。
- 当前集没有分镜脚本：视频提示词生成按钮不可用，提示先拆解分镜。
- `章节导演板` 没有故事板：故事板区域显示生成入口，不自动进入视频提示词。
- `首帧衔接` 被选中：所有相关生成动作显示暂不可用。
- 后端提示词推理返回空镜头或空内容时，前端应展示具体 Clip 编号和失败原因。

## 测试设计

### 后端 fixture：门缝丧尸事件顺序

输入包含用户提供的原文片段：

- 枪口火舌；
- “砰”；
- 门缝丧尸被击中；
- Flora 随后说“杀戮！这是赤裸裸的杀戮！”。

断言：

- beat 顺序是门缝丧尸探头 → Chloe 开枪/命中 → Flora 训斥；
- Flora 的对白不出现在枪响之前；
- 视频提示词中门缝位置没有丢失；
- 失败输出包含源文本片段、期望顺序、实际顺序和 prompt 摘录。

### 后端 fixture：资产图物理约束

输入角色资产事实：角色戴头盔。原文没有摘头盔动作。

断言：

- 视频提示词不能出现摸头发、甩头发、抓头发等与头盔冲突的动作；
- 如果原文明确摘头盔，则后续状态账本显示未戴头盔；
- 状态恢复前，后续 clip prompt 不得默认回到戴头盔状态。

### 后端 fixture：状态延续

输入分镜：

- beat 1：角色扔掉枪；
- beat 2：角色继续移动；
- clip 2：角色说话。

断言：

- beat 2 和 clip 2 都明确角色未持枪；
- 没有“手持枪”回弹；
- 如果后续 beat 写明捡回枪，状态才恢复为持枪。

### 前端/UI 测试

断言：

- `Seedance 多参` 不显示故事板图、故事板提示词、故事板生图入口。
- `章节导演板` 显示故事板区域。
- 顶部 `全流程推理`、`更多操作` 不再出现。
- `首帧衔接` 显示不可用状态。
- 视频提示词按钮在缺资产或缺分镜时有明确不可用文案。

- 目标后端测试
- `npm run build`

## 验收标准

1. 用户在 `Seedance 多参` 下看不到故事板相关页面。
2. 用户只看到清晰的三步主动作，不再看到重复的全流程按钮。
3. 重新拆解分镜不会直接生成视频提示词或写画布视频节点。
4. 生成视频提示词会读取资产、分镜、全局设定和状态账本。
5. 门缝丧尸例子按正确事件顺序输出。
6. 对白不提前、不跨 Clip 截断。
7. 每个 Clip 的视频 prompt 明确角色位置、朝向、状态、持物、穿戴。
8. 状态跨 beat/clip 延续，直到文本明确变化。
9. 资产图约束被遵守，不产生物理冲突动作。
10. 当前项目 prompt 相关全局设定已翻译成英文，项目名仍为中文。
11. 测试失败信息足够直观，能定位原文、期望、实际和 prompt 问题。

## 实施顺序建议

1. 前端先收束策略枚举和按钮显示。
2. 后端补强分镜拆解 prompt，使事件顺序、对白时机、镜头内容和 1-3 秒节奏更明确。
3. 后端加入资产约束和状态账本到视频提示词生成。
4. 按策略调整画布同步，区分 `Seedance 多参` 与 `章节导演板`。
5. 增加直观 fixture 测试。
6. 做当前项目全局设定英文翻译。
7. 跑完整验证命令并手动检查一组真实 Clip。

## 风险

- 模型仍可能忽略部分推理规则，所以需要代码层的结构化检查和测试 fixture 兜底。
- 状态账本如果只靠自由文本解析会不稳定，实施时应尽量把状态写成结构化字段再组装 prompt。
- 隐藏故事板区域会改变用户已有习惯，需要文案明确当前策略为什么不需要故事板。
- 当前工作区已有大量未提交改动，实施时必须只处理本任务相关文件，不回滚无关变更。
